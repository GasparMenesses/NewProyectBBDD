# Informe Individual - Gonzalo Di Cono

Este proyecto me permitió aplicar en la práctica varios conceptos clave de desarrollo de software, especialmente la integración entre base de datos, backend y frontend.

# Tarea principal
Mi principal participación estuvo orientada a la implementación de la arquitectura del proyecto con Docker. Además, trabajé en el diseño de la base de datos, la creación de tablas y el desarrollo de consultas SQL para cargar y consultar datos.

También puse mi granito de arena en el desarrollo del backend en Python y participé en el frontend, realizando pequeños ajustes de interfaz y agregando controles de validación.

# Dificultades
La parte que más me costó fue el backend, ya que hasta ahora no habíamos visto HTML ni CSS en ninguna materia, ni habíamos trabajado con frameworks de frontend o con Flask en el curso. Entender cómo conectar la base de datos con el backend y cómo exponer los datos para el frontend fue un desafío importante. 

# Conclusión
Me resultó un proyecto muy interesante porque aprendí cómo conectar una base de datos con el backend y el frontend, y cómo estos componentes pueden convivir dentro de un contenedor usando Docker Compose.
Quiero destacar el buen trabajo y funcionamiento del equipo. Siempre estuvieron disponibles y fueron un gran apoyo cada vez que surgía alguna dificultad o duda.
